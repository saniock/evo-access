# Страница Audit

Журнал всех изменений в системе доступов. Каждое изменение — создание/редактирование/удаление роли, назначение/снятие роли с менеджера, добавление/снятие override — записывается в лог.

## Зачем это нужно

1. **Безопасность** — можно увидеть, кто и когда изменил чьи-то права. Если менеджер получил доступ, которого не должен был — в логе видно, кто ему дал.
2. **Отладка** — если менеджер жалуется, что "вчера работало, а сегодня нет" — в журнале видно, что изменилось.
3. **Отчётность** — полная история действий с правами.

## Колонки журнала

- **Date/Time** — когда произошло
- **Actor** — кто сделал изменение (ID менеджера)
- **Action** — тип действия:
  - `grant` — добавлено разрешение роли
  - `revoke` — снято разрешение роли
  - `user_assigned` — менеджеру назначена роль
  - `user_role_changed` — у менеджера изменена роль
  - `role_created`, `role_deleted`, `role_cloned` — CRUD ролей
  - `override_grant`, `override_revoke`, `override_removed` — изменения override
- **Role ID** — ID роли (если действие касается роли)
- **User ID** — ID менеджера (если действие касается менеджера)
- **Perm ID** — ID разрешения
- **Old / New** — старое и новое значения (для действий, где это имеет смысл)
- **Details** — дополнительные данные в JSON-формате

## Фильтры

Панель фильтров позволяет сузить журнал:

- **From / To** — диапазон дат
- **Actor ID** — только действия конкретного менеджера
- **Action type** — только один тип действия

Нажмите **Filter**, чтобы применить. Без фильтров показываются последние 500 записей.

## Невозможность редактирования

Журнал — **только для чтения**. Его нельзя очистить через интерфейс — это сделано намеренно, чтобы никто не мог замести следы.
