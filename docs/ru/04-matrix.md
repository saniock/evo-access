# Страница Matrix

Управление разрешениями **ролей**. Здесь вы выбираете роль и расставляете ей права на все модули сайта.

## Интерфейс

Слева — **список всех ролей** с количеством назначенных менеджеров в скобках. Справа — **таблица разрешений** выбранной роли.

Кликните на роль в левом списке — справа появится полная таблица всех разрешений системы, сгруппированная по модулям. Каждая строка — одно разрешение (например, `orders.orders`). Колонки — действия (view, edit, delete, import и т.д.).

## Управление разрешениями

Кликните по **бейджу** в ячейке, чтобы переключить разрешение:

- **Серый —** клик → **Зелёный ✓** (добавить разрешение)
- **Зелёный ✓** клик → **Серый —** (забрать разрешение)

Изменения **сохраняются автоматически** — не нужно нажимать кнопку Save. Каждый клик — это один запрос к серверу, который сразу обновляет базу.

## Что означает добавление разрешения

Если вы добавляете разрешение роли `seller` на `orders.orders.view`, то **все менеджеры** с этой ролью сразу получают право просматривать заказы. В следующий раз, когда менеджер обновит страницу — он увидит раздел *Orders*.

## Что означает снятие разрешения

Если вы снимаете разрешение, все менеджеры с этой ролью **теряют доступ** к разделу. Исключения — менеджеры, у которых есть **override-grant** на это же действие (они сохранят доступ персонально).

## Системные роли

Если вы выбрали системную роль (например, `superadmin`) — таблица будет доступна только для чтения. Системная роль всегда имеет **полный доступ** и его нельзя изменить через матрицу. Это сделано для безопасности — чтобы случайно не лишить самого себя доступа.

## Когда использовать матрицу, а когда Users

- **Matrix** — когда меняете права, которые должны касаться **всех** менеджеров с ролью. Например, добавляете новый модуль и хотите, чтобы все продавцы его видели.
- **Users** — когда одному конкретному менеджеру нужно дать/забрать что-то. В этом случае его роль не меняется — вы добавляете override поверх неё.
