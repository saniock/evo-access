# Страница Users

Это главная страница для управления правами конкретных менеджеров. Тут вы видите всех менеджеров сайта и можете назначать им роли и точечные права.

## Таблица менеджеров

Каждая строка — один менеджер. Колонки:

- **ID** — идентификатор менеджера в системе
- **Manager** — имя из профиля EVO
- **Role** — текущая роль (или пусто, если ещё не назначена)
- **Modules** — модули, к которым менеджер имеет хоть какой-то доступ
- **Grants** — полоска прогресса: сколько действий ему доступно из всех возможных
- **Overrides** — количество точечных разрешений/запретов поверх роли
- **Edit** — кнопка открытия полной матрицы прав

## Открытие матрицы менеджера

Кликните кнопку **Edit** или дважды кликните по строке. Откроется popup с полной матрицей разрешений менеджера, сгруппированной по модулям.

### Изменение роли

В верхней части popup есть выпадающий список **Role**. Выберите новую роль — она применится после нажатия **Save**.

### Бейджи разрешений

Каждая ячейка матрицы — это **бейдж** одного из четырёх цветов:

- **Зелёный ✓** — действие разрешено **от роли** (унаследовано)
- **Синий ✓** — действие разрешено **через override** (добавлено лично этому менеджеру)
- **Красный ✗** — действие **запрещено через override** (снято лично у этого менеджера, даже если роль разрешает)
- **Серый —** — нет доступа

### Как менять права

Кликайте по бейджам, чтобы переключать состояние:

- **Серый → Синий** — добавить индивидуальное разрешение
- **Зелёный → Красный** — забрать разрешение, которое даёт роль
- **Синий → Серый** — убрать добавленное индивидуальное разрешение
- **Красный → Зелёный** — вернуть разрешение роли

После всех изменений нажмите **Save**. Все override сохранятся одним пакетом.

## Аккордеон модулей

В матрице модули развёрнуты автоматически, если в них есть хотя бы одно разрешение у менеджера. Модули без доступа — свёрнуты, чтобы не занимать место. Кликните на заголовок модуля, чтобы развернуть/свернуть его.

## Назначение роли новому менеджеру

Если у менеджера ещё нет роли, в списке он показан с пустым полем **Role**. Откройте его матрицу, выберите роль из списка, нажмите **Save**. Роль назначится — менеджер получит все разрешения этой роли.
