# Сторінка Matrix

Керування дозволами **ролей**. Тут ви вибираєте роль і розставляєте їй права на всі модулі сайту.

## Інтерфейс

Зліва — **список усіх ролей** з кількістю призначених менеджерів у дужках. Справа — **таблиця дозволів** обраної ролі.

Клацніть на роль у лівому списку — праворуч з'явиться повна таблиця всіх дозволів системи, згрупована по модулях. Кожен рядок — один дозвіл (наприклад, `orders.orders`). Колонки — дії (view, edit, delete, import тощо).

## Керування дозволами

Клацніть по **бейджу** у клітинці, щоб перемкнути дозвіл:

- **Сірий —** клацнути → **Зелений ✓** (додати дозвіл)
- **Зелений ✓** клацнути → **Сірий —** (забрати дозвіл)

Зміни **зберігаються автоматично** — не потрібно натискати кнопку Save. Кожен клік — це один запит до сервера, який одразу оновлює базу.

## Що означає додавання дозволу

Якщо ви додаєте дозвіл ролі `seller` на `orders.orders.view`, то **всі менеджери** з цією роллю відразу отримують право переглядати замовлення. Наступного разу, коли менеджер оновить сторінку — він побачить розділ *Orders*.

## Що означає забирання дозволу

Якщо ви забираєте дозвіл, усі менеджери з цією роллю **втрачають доступ** до розділу. Винятки — менеджери, які мають **override-grant** на цю ж дію (вони збережуть доступ персонально).

## Системні ролі

Якщо ви обрали системну роль (наприклад, `superadmin`) — таблиця буде доступна тільки для перегляду. Системна роль завжди має **повний доступ** і його не можна змінити через матрицю. Це зроблено для безпеки — щоб випадково не позбавити самого себе доступу.

## Коли використовувати матрицю, а коли Users

- **Matrix** — коли змінюєте права, які мають стосуватися **всіх** менеджерів з роллю. Наприклад, додаєте новий модуль і хочете, щоб усі продавці його бачили.
- **Users** — коли одному конкретному менеджеру треба дати/забрати щось. У цьому випадку його роль не змінюється — ви додаєте override поверх неї.
